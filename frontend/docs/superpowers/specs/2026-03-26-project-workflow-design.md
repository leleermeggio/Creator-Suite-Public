# CazZone — Project-Based Creator Workflow

## Overview

Transform CazZone from a standalone tool launcher into a **project-based workspace** for content creators. Users create projects with defined workflow phases, and the app guides them through each phase — suggesting the right tools at the right moment while keeping all tools accessible at any time.

**Approach:** Guided Workspace (Hybrid) — structured phases with full tool freedom. GPS-style: suggests the route, but you can take detours.

## Project Creation

Two paths to create a new project:

### Path 1: Template Picker (fast)

Pre-built templates the user selects and optionally customizes:

| Template | Phases |
|----------|--------|
| **🎬 YouTuber** | Registra → Taglia → Edita → Pubblica |
| **🎙️ Podcaster** | Registra → Trascrivi → Taglia → Export |
| **📱 Content Creator** | Idea → Registra → Short Edit → Multi-post |
| **✏️ Personalizzato** | User defines phases manually |

After selecting a template, the user names the project and can add/remove/reorder phases before starting.

### Path 2: AI Chat Setup (flexible)

User describes their workflow in natural language:
> "Sono uno YouTuber che parla di calcio, registro video reaction post-partita di 40-60 min, poi taglio i momenti morti e aggiungo editing..."

The app sends this to Gemini AI (already available via `GOOGLE_API_KEY`) to generate:
- Project name suggestion
- Ordered phases with icons and suggested tools per phase
- User reviews and edits before confirming

**AI Setup — Gemini Integration:**

- **Model:** `gemini-2.0-flash` via REST API (`https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent`). Model version configurable in Settings.
- **Auth:** `GOOGLE_API_KEY` from Settings (user enters it in-app, stored in AsyncStorage)
- **Fallback:** If no API key or Gemini fails, show a toast and redirect to the manual "Personalizzato" template flow

**System prompt:**
```
You are a workflow assistant for content creators. The user will describe their creative process.
Return ONLY valid JSON with this structure:
{
  "projectName": "suggested name",
  "description": "one-line summary",
  "phases": [
    {
      "name": "phase name in Italian",
      "icon": "single emoji",
      "suggestedToolIds": ["toolId1", "toolId2"]
    }
  ]
}
Available tool IDs: transcribe, translate, download, summarize, ocr, tts, convert, jumpcut, ai-image
Return 3-6 phases. Keep phase names short (1-2 words, Italian).
```

The user's natural language input is the user message.

## Project Workspace (Tab Phases)

### Layout

When a project is open, the screen shows:

1. **Project header** — name, progress indicator (e.g., "Phase 2 of 4")
2. **Phase tab bar** — horizontal, swipeable tabs. Active phase highlighted with its accent color. Completed phases show ✅. Upcoming phases dimmed.
3. **Phase content area** (below tabs):
   - **Files in this phase** — list of files (uploaded or generated by tools). Each file shows name, type, size, source.
   - **Suggested next step** — prominent CTA card highlighting the most relevant tool for this phase (e.g., "Jump Cut" in the Taglia phase). One tap to launch.
   - **Other available tools** — pill buttons for all 9 tools. Suggested ones first, then the rest under "+ Tutti".

### Phase Lifecycle

```
PENDING → ACTIVE → COMPLETED
```

- A phase becomes **ACTIVE** when the user taps on it or when the previous phase is completed.
- A phase is **COMPLETED** when the user explicitly marks it done (button) or advances to the next phase.
- Users CAN go back to completed phases to re-process or add files.
- Users CAN skip ahead to any phase (the guidance is suggestions, not walls).

### File Flow Between Phases

When a tool produces output in a phase:
- The output file is saved to the **current phase**.
- The app offers to **link it to the next phase** as input (one-tap "Invia a fase successiva"). This creates a new `ProjectFile` entry in the next phase pointing to the same `localUri` — no file duplication on disk.
- Files can be manually moved between phases via long-press → "Sposta a...".

5. For Cover Social: "Genera tutti i formati" button generates all platform sizes in one batch

**In project workflow:** Suggested in **Editing** and **Publish** phases. Always accessible from any phase and Quick Tools.

## Tool Execution Architecture

Tools in v1 run **entirely client-side within the Expo app.** There is no backend API or connection to the Telegram bot.

| Tool | Client-side Implementation |
|------|---------------------------|
| `transcribe` | Deferred to v2 (requires Whisper backend). Shows "Coming soon — use Telegram bot" in v1. |
| `translate` | If `GOOGLE_API_KEY` set: Gemini API translation. Fallback: free `api.mymemory.translated.net` (no key needed, 5000 chars/day). |
| `download` | Not feasible client-side due to platform restrictions. Shows "Coming soon — use Telegram bot" in v1. |
| `summarize` | Gemini API call via `GOOGLE_API_KEY` (same as AI setup) |
| `ocr` | Gemini API call with image attachment (Gemini Vision) |
| `tts` | Deferred to v2. Shows "Coming soon — use Telegram bot" in v1. |
| `convert` | Deferred to v2. Shows "Coming soon — use Telegram bot" in v1. |
| `jumpcut` | Deferred to v2. Shows "Coming soon — use Telegram bot" in v1. |
| `ai-image` | Pollinations.ai HTTP GET (fully client-side, no key) |

**v1 available tools (4 of 9):** translate, summarize, ocr, ai-image. The remaining 5 show a clear "coming soon" state with a prompt to use the Telegram bot. All 9 appear in the UI for completeness.

**Tool execution flow:**
1. User taps tool → tool screen opens with project/phase context
2. User provides input (text, file, or URL)
3. App calls the relevant API
4. Result displayed inline
5. User taps "Salva" → file written to `FileSystem.documentDirectory/projects/{projectId}/{phaseId}/`
6. `ProjectFile` entry added to project metadata in storage

## File Upload

Files are added to phases via:

- **Native (iOS/Android):** `expo-document-picker` for documents/video/audio. `expo-image-picker` for camera roll photos.
- **Web:** Standard `<input type="file">` wrapped in a `Pressable` drop zone. Also supports drag-and-drop on the drop zone.
- **Upload button** appears in each phase content area as a "+ Aggiungi file" card.
- Uploaded files are copied to `FileSystem.documentDirectory/projects/{projectId}/{phaseId}/{filename}`.
- Max file size: 500 MB (show warning above this; app may slow down).

## Activity Queue (Attività Tab)

The Attività tab shows a chronological list of tool execution jobs.

### Job Model

```typescript
interface Job {
  id: string;
  toolId: string;
  projectId?: string;         // null for Quick Tools jobs
  phaseId?: string;
  status: 'running' | 'completed' | 'failed';
  inputSummary: string;       // e.g., "Thumbnail: Roma vs Juve reaction"
  outputFileId?: string;      // link to ProjectFile if saved
  startedAt: string;
  completedAt?: string;
  error?: string;
}
```

### Behavior
- A `Job` is created every time a tool starts processing.
- Jobs are stored in `AsyncStorage` key `jobs` (capped at last 100 entries, older ones pruned).
- The Attività tab groups by: **In corso** (running) at top, then **Recenti** (completed/failed) below.
- Each job card shows: tool icon, input summary, project name (if any), status, timestamp.
- Tapping a completed job navigates to the project/phase where the output was saved.
- **Note:** In v1, all API calls are short-lived (< 30 seconds). There is no background processing. "Running" means the user is on the tool screen waiting. The Attività tab is primarily a history log. Background processing is out of scope for v1.

## Destructive Actions

| Action | UX |
|--------|----|
| **Delete project** | Long-press project card → "Elimina progetto" option. Confirmation dialog: "Eliminare {name} e tutti i file? Questa azione è irreversibile." Deletes project metadata + all files in project directory. |
| **Delete file** | Swipe-left on file row (native) or three-dot menu (web) → "Elimina". No confirmation for single files. |

**Project list sections:** Active (default view), Archived (collapsible section at bottom), Completed (shown with ✅ badge in the active list, filterable).

## Custom Template Builder ("Personalizzato")

When user picks "Personalizzato" in the template picker:

1. **Name screen** — text input for project name
2. **Phase builder** — starts with one empty phase slot
   - Each phase has: text input (name), emoji picker (icon), color picker (6 preset neon colors)
   - "+ Aggiungi fase" button to add more
   - Drag handles to reorder phases
   - Swipe-left to delete a phase
   - Suggested tools auto-populated: empty by default, user can optionally tap "+ Strumenti" to pick suggested tools from a checkbox list of all 9
3. **Review screen** — shows the full pipeline visually, "Crea progetto" button

## AI Image Batch Generation

When "Genera tutti i formati" is tapped in the Cover Social mode:

1. All 4 platform variants are generated **sequentially** (one at a time) to avoid overwhelming the API.
2. A progress indicator shows "Generando 1 di 4... YouTube" → "2 di 4... TikTok" etc.
3. If one fails, it's marked with a retry button; the batch continues with the remaining formats.
4. Results appear in a 2x2 grid preview. Each can be individually saved or regenerated.
5. "Salva tutti" button saves all successful images to the current phase.

## Storage

All data stored locally on device using **AsyncStorage** and **Expo FileSystem**. No backend required for v1 — local-first app.

### Metadata (AsyncStorage)

Each project stored under its own key to avoid the 6MB single-key limit on Android:

| Key Pattern | Value |
|-------------|-------|
| `project:{id}` | Single `Project` JSON object |
| `project_index` | Array of `{ id, name, status, updatedAt }` for list rendering |
| `custom_templates` | Array of user-created `Template` objects |
| `jobs` | Array of last 100 `Job` objects |
| `settings` | App settings (API keys, preferences) |

### Files (Expo FileSystem)

```
FileSystem.documentDirectory/
  projects/
    {projectId}/
      {phaseId}/
        {filename}
```

### Built-in Templates

Hardcoded in `constants/templates.ts`, not stored in AsyncStorage.
